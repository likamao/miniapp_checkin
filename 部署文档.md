# 打卡系统部署文档

## 1. 环境准备

### 1.1 系统要求
- JDK 11+
- MySQL 8.0+
- Maven 3.6+
- 微信开发者工具

### 1.2 数据库准备
1. 登录MySQL数据库
2. 创建数据库：`CREATE DATABASE checkin CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`
3. 执行数据库初始化脚本：`source /path/to/schema.sql`
4. 执行RBAC权限初始化脚本：`source /path/to/rbac_ddl.sql` 和 `source /path/to/rbac_init.sql`

## 2. 后端服务部署

### 2.1 配置修改
1. 编辑 `checkin-backend/src/main/resources/application.yml` 文件
2. 修改数据库配置：
   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://localhost:3306/checkin?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
       username: root
       password: 123456
   ```
3. 修改微信配置：
   ```yaml
   wechat:
     appid: your_appid
     secret: your_secret
   ```

### 2.2 构建和运行
1. 进入后端项目目录：`cd checkin-backend`
2. 构建项目：`mvn clean package`
3. 运行项目：`java -jar target/checkin-backend-1.0-SNAPSHOT.jar`

## 3. 微信小程序配置

### 3.1 小程序注册
1. 登录微信公众平台（https://mp.weixin.qq.com/）
2. 注册小程序账号
3. 获取AppID和AppSecret

### 3.2 小程序配置
1. 编辑 `checkin-miniprogram/app.js` 文件
2. 修改后端API地址：
   ```javascript
   wx.request({
     url: 'http://localhost:8080/api/auth/login',
     // ...
   });
   ```

### 3.3 小程序上传和发布
1. 使用微信开发者工具打开 `checkin-miniprogram` 目录
2. 上传代码到微信公众平台
3. 提交审核并发布

## 4. 系统测试

### 4.1 API测试
使用Postman或curl测试后端API接口：
1. 微信登录接口：`POST http://localhost:8080/api/auth/login`
2. 创建打卡记录：`POST http://localhost:8080/api/checkin/create`
3. 获取打卡记录：`GET http://localhost:8080/api/checkin/records`
4. 获取统计数据：`GET http://localhost:8080/api/checkin/statistics`

### 4.2 小程序测试
1. 在微信开发者工具中预览小程序
2. 测试登录功能
3. 测试打卡功能
4. 测试记录查看功能
5. 测试统计展示功能

## 5. 常见问题

### 5.1 数据库连接失败
- 检查MySQL服务是否启动
- 检查数据库配置是否正确
- 检查数据库用户权限

### 5.2 微信登录失败
- 检查AppID和AppSecret是否正确
- 检查小程序是否已发布
- 检查网络连接

### 5.3 打卡提交失败
- 检查后端服务是否运行
- 检查token是否有效
- 检查表单数据是否符合要求

## 6. 系统维护

### 6.1 日志管理
- 后端日志位于 `logs` 目录
- 小程序日志可在微信开发者工具中查看

### 6.2 数据备份
- 定期备份MySQL数据库
- 备份 `checkin` 数据库

### 6.3 版本更新
- 后端代码更新后重新构建和部署
- 小程序代码更新后重新上传和发布
