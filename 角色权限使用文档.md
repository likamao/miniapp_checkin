# 打卡系统角色权限使用文档

## 1. 文档概述

### 1.1 文档目的
本文档详细描述打卡系统的角色权限管理机制，包括系统中的所有用户角色、各角色对应的权限范围、权限分配流程、权限变更记录以及权限管理相关的操作指南，为系统管理员和开发人员提供完整的权限管理参考。

### 1.2 适用范围
本文档适用于打卡系统的管理员、开发人员以及需要了解系统权限管理机制的相关人员。

## 2. 角色权限体系

### 2.1 角色定义
系统采用基于角色的访问控制（RBAC）机制，包含以下角色：

| 角色名称 | 角色代码 | 描述 |
| --- | --- | --- |
| 管理员 | ADMIN | 拥有系统所有权限，可管理用户、角色和权限 |
| 发布者 | PUBLISHER | 拥有部分管理权限，可查看用户信息和管理打卡记录 |
| 查看者 | VIEWER | 可查看公开主题的完整打卡记录，拥有基础打卡权限 |
| 普通用户 | USER | 拥有基础打卡权限，可进行日常打卡操作 |

### 2.2 权限定义
系统包含以下权限：

| 权限名称 | 权限代码 | 描述 |
| --- | --- | --- |
| 用户查询权限 | user:read | 查看用户信息的权限 |
| 用户创建权限 | user:create | 创建用户的权限 |
| 用户更新权限 | user:update | 更新用户信息的权限 |
| 用户删除权限 | user:delete | 删除用户的权限 |
| 打卡查询权限 | checkin:read | 查看打卡记录的权限 |
| 打卡详情权限 | checkin:view-details | 查看打卡详情（包括标题和内容）的权限 |
| 打卡创建权限 | checkin:create | 创建打卡记录的权限 |
| 打卡统计权限 | checkin:statistics | 查看打卡统计的权限 |

### 2.3 角色权限映射
各角色对应的权限范围如下：

| 角色 | 权限范围 |
| --- | --- |
| ADMIN | user:read, user:create, user:update, user:delete, checkin:read, checkin:view-details, checkin:create, checkin:statistics |
| PUBLISHER | user:read, checkin:read, checkin:view-details, checkin:create, checkin:statistics |
| VIEWER | checkin:read, checkin:create, checkin:statistics, checkin:view-details |
| USER | checkin:read, checkin:create, checkin:statistics |

## 3. 权限分配流程

### 3.1 初始化权限
系统首次部署时，通过执行 `rbac_init.sql` 脚本初始化角色和权限数据：

1. 插入系统预设角色（ADMIN、PUBLISHER、VIEWER、USER）
2. 插入系统预设权限
3. 为各角色分配对应的权限

### 3.2 用户角色分配
为用户分配角色的流程：

1. 管理员登录系统
2. 进入用户管理界面
3. 选择目标用户
4. 为用户分配适当的角色
5. 保存角色分配结果

### 3.3 权限变更流程
修改角色权限的流程：

1. 管理员登录系统
2. 进入角色管理界面
3. 选择目标角色
4. 修改角色的权限分配
5. 保存权限变更结果

## 4. 权限变更记录

### 4.1 变更记录格式
权限变更记录应包含以下信息：
- 变更时间
- 变更类型（新增/修改/删除）
- 变更内容
- 变更人
- 变更原因

### 4.2 变更记录示例

| 变更时间 | 变更类型 | 变更内容 | 变更人 | 变更原因 |
| --- | --- | --- | --- | --- |
| 2026-03-17 | 新增 | 为VIEWER角色添加checkin:view-details权限 | 系统管理员 | 提升查看者角色的权限范围 |
| 2026-03-18 | 修改 | 调整PUBLISHER角色的权限列表 | 系统管理员 | 优化角色权限结构 |

## 5. 权限管理操作指南

### 5.1 添加管理员角色
系统部署后，可通过执行 `add_admin_role.sql` 脚本为指定用户添加管理员角色：

1. 编辑 `add_admin_role.sql` 文件，修改目标用户的openid
2. 执行脚本：`source /path/to/add_admin_role.sql`
3. 验证用户是否成功获得管理员权限

### 5.2 查看用户权限
通过以下步骤查看用户的权限：

1. 管理员登录系统
2. 进入用户管理界面
3. 选择目标用户
4. 查看用户的角色和权限信息

### 5.3 修改角色权限
通过以下步骤修改角色的权限：

1. 管理员登录系统
2. 进入角色管理界面
3. 选择目标角色
4. 勾选或取消勾选相应的权限
5. 保存修改结果

### 5.4 新增角色
通过以下步骤新增角色：

1. 管理员登录系统
2. 进入角色管理界面
3. 点击"新增角色"按钮
4. 填写角色名称和描述
5. 为新角色分配适当的权限
6. 保存角色信息

### 5.5 删除角色
通过以下步骤删除角色：

1. 管理员登录系统
2. 进入角色管理界面
3. 选择目标角色
4. 点击"删除"按钮
5. 确认删除操作

## 6. 权限验证机制

### 6.1 请求验证流程
系统通过 `PermissionInterceptor` 拦截器对API请求进行权限验证：

1. 请求到达系统
2. 拦截器提取JWT令牌
3. 解析令牌获取用户信息
4. 查询用户的角色和权限
5. 验证用户是否拥有访问当前API的权限
6. 允许或拒绝请求

### 6.2 权限验证规则
- 所有API接口均需验证用户身份
- 不同接口需要不同的权限级别
- 管理员拥有所有接口的访问权限
- 普通用户只能访问与自身相关的接口

## 7. 常见问题与解决方案

### 7.1 权限不足问题
**问题**：用户访问接口时提示权限不足

**解决方案**：
1. 检查用户的角色分配
2. 确认角色是否拥有相应的权限
3. 如需要，为用户分配更高级别的角色或为角色添加相应的权限

### 7.2 角色分配失败
**问题**：无法为用户分配角色

**解决方案**：
1. 检查数据库连接是否正常
2. 确认用户和角色是否存在
3. 检查SQL执行权限

### 7.3 权限冲突
**问题**：用户同时拥有多个角色，权限出现冲突

**解决方案**：
1. 检查角色权限分配是否合理
2. 调整角色权限结构，避免权限冲突
3. 为用户分配最合适的单一角色

## 8. 安全注意事项

### 8.1 权限管理安全
- 管理员账号应妥善保管，避免泄露
- 定期审查角色权限分配，确保权限合理
- 及时撤销不再需要的权限

### 8.2 数据安全
- 权限变更操作应记录日志
- 敏感操作应进行二次确认
- 定期备份权限相关数据

## 9. 总结

本文档详细描述了打卡系统的角色权限管理机制，包括角色定义、权限分配、操作流程等内容。通过合理的权限管理，可以确保系统的安全性和稳定性，同时为不同用户提供适当的访问权限。系统管理员应定期审查和更新权限设置，以适应业务需求的变化。
