# 打卡系统项目需求文档

## 1. 项目概述

### 1.1 项目背景
随着移动互联网的普及，打卡系统已成为企业管理、个人习惯养成的重要工具。本项目旨在开发一个基于微信小程序的打卡系统，方便用户进行日常打卡记录，并提供统计分析功能，帮助用户养成良好的打卡习惯。

### 1.2 项目目标
- 开发一个功能完善的微信小程序打卡系统
- 实现用户微信登录认证
- 提供打卡记录的创建、查询功能
- 实现打卡数据的统计分析
- 确保系统的安全性和稳定性

### 1.3 项目范围
本项目包括后端API服务和前端微信小程序两部分：
- 后端：基于Spring Boot框架，提供RESTful API接口
- 前端：基于微信小程序框架，实现用户界面和交互逻辑

## 2. 核心功能模块

### 2.1 认证模块
- **微信登录**：用户通过微信小程序登录，获取JWT令牌
- **用户信息管理**：更新用户昵称、头像等信息

### 2.2 打卡模块
- **创建打卡记录**：用户提交打卡标题和内容
- **查询打卡记录**：支持按年、月查询打卡记录，支持分页
- **今日打卡状态检查**：检查用户当天是否已经打卡

### 2.3 统计模块
- **最近7天打卡统计**：统计用户最近7天的打卡次数
- **月度打卡统计**：按周统计用户月度打卡情况

## 3. 用户角色与权限

### 3.1 用户角色
本系统采用基于角色的访问控制（RBAC）机制，包含以下角色：
- **普通用户**：通过微信登录的常规用户
- **管理员**：具有系统管理权限的用户

### 3.2 权限定义
- **登录权限**：所有微信用户均可登录
- **打卡权限**：登录用户可创建打卡记录
- **查询权限**：登录用户可查询自己的打卡记录和统计数据
- **统计权限**：登录用户可查看自己的打卡统计数据
- **管理权限**：管理员可管理用户、角色和权限

### 3.3 RBAC权限控制
系统通过以下表结构实现RBAC权限控制：
- **角色表**：存储系统角色信息
- **权限表**：存储系统权限信息
- **用户角色关联表**：建立用户与角色的关联关系
- **角色权限关联表**：建立角色与权限的关联关系

## 4. 业务流程

### 4.1 用户登录流程
1. 用户打开微信小程序
2. 小程序获取微信登录code
3. 调用后端登录接口，传入code
4. 后端通过code获取微信用户信息，创建或更新用户记录
5. 后端生成JWT令牌并返回给前端
6. 前端存储令牌，完成登录

### 4.2 打卡流程
1. 用户进入打卡页面
2. 系统检查用户登录状态
3. 系统检查用户当天是否已打卡
4. 用户填写打卡标题和内容
5. 用户提交打卡信息
6. 后端创建打卡记录并更新统计数据
7. 前端显示打卡成功提示和庆祝动画

### 4.3 查看记录流程
1. 用户进入记录页面
2. 系统检查用户登录状态
3. 系统加载用户打卡记录
4. 用户可选择按年、月筛选记录
5. 系统分页显示打卡记录

### 4.4 查看统计流程
1. 用户进入统计页面
2. 系统检查用户登录状态
3. 系统加载用户最近7天打卡统计
4. 系统加载用户月度打卡统计

## 5. 界面交互规范

### 5.1 登录页面
- 微信一键登录按钮
- 登录成功后自动跳转到打卡页面

### 5.2 打卡页面
- 打卡标题输入框（最多50字）
- 打卡内容输入框（最多500字）
- 提交按钮
- 已打卡状态提示
- 鼓励性文本显示
- 打卡成功后的庆祝动画

### 5.3 记录页面
- 年份、月份选择器
- 打卡记录列表（包含标题、内容、打卡时间）
- 分页加载

### 5.4 统计页面
- 最近7天打卡次数显示
- 月度打卡统计图表
- 按周显示打卡次数

## 7. 接口说明

### 7.1 认证接口

#### 7.1.1 微信登录
- **接口地址**: `/api/auth/login`
- **请求方法**: POST
- **请求参数**:
  | 参数名 | 类型 | 必选 | 描述 |
  | --- | --- | --- | --- |
  | code | string | 是 | 微信登录code |
- **响应格式**:
  ```json
  {
    "token": "JWT token",
    "user": {
      "id": 1,
      "openid": "xxx",
      "nickname": "xxx",
      "avatarUrl": "xxx"
    }
  }
  ```

#### 7.1.2 更新用户信息
- **接口地址**: `/api/auth/updateInfo`
- **请求方法**: POST
- **请求参数**:
  | 参数名 | 类型 | 必选 | 描述 |
  | --- | --- | --- | --- |
  | userId | string | 是 | 用户ID |
  | nickname | string | 是 | 用户昵称 |
  | avatarUrl | string | 是 | 头像URL |
- **响应格式**:
  ```json
  {
    "user": {
      "id": 1,
      "openid": "xxx",
      "nickname": "xxx",
      "avatarUrl": "xxx"
    }
  }
  ```

### 7.2 打卡接口

#### 7.2.1 创建打卡记录
- **接口地址**: `/api/checkin/create`
- **请求方法**: POST
- **请求头**:
  | 头部名 | 值 |
  | --- | --- |
  | Authorization | Bearer {token} |
- **请求参数**:
  | 参数名 | 类型 | 必选 | 描述 |
  | --- | --- | --- | --- |
  | title | string | 是 | 打卡标题（50字以内） |
  | content | string | 是 | 打卡内容（500字以内） |
- **响应格式**:
  ```json
  {
    "record": {
      "id": 1,
      "title": "xxx",
      "content": "xxx",
      "checkinTime": "2026-02-28 16:00:00"
    }
  }
  ```

#### 7.2.2 获取打卡记录列表
- **接口地址**: `/api/checkin/records`
- **请求方法**: GET
- **请求头**:
  | 头部名 | 值 |
  | --- | --- |
  | Authorization | Bearer {token} |
- **请求参数**:
  | 参数名 | 类型 | 必选 | 描述 |
  | --- | --- | --- | --- |
  | year | int | 否 | 年份 |
  | month | int | 否 | 月份 |
  | page | int | 否 | 页码（默认1） |
  | pageSize | int | 否 | 每页大小（默认10） |
- **响应格式**:
  ```json
  {
    "records": [
      {
        "id": 1,
        "title": "xxx",
        "content": "xxx",
        "checkinTime": "2026-02-28 16:00:00",
        "createdAt": "2026-02-28 16:00:00",
        "updatedAt": "2026-02-28 16:00:00"
      }
    ],
    "pagination": {
      "total": 10,
      "totalPages": 1,
      "currentPage": 1,
      "pageSize": 10
    }
  }
  ```

#### 7.2.3 获取打卡统计数据
- **接口地址**: `/api/checkin/statistics`
- **请求方法**: GET
- **请求头**:
  | 头部名 | 值 |
  | --- | --- |
  | Authorization | Bearer {token} |
- **响应格式**:
  ```json
  {
    "recent7DaysCount": 5
  }
  ```

#### 7.2.4 检查今日打卡状态
- **接口地址**: `/api/checkin/check-today`
- **请求方法**: GET
- **请求头**:
  | 头部名 | 值 |
  | --- | --- |
  | Authorization | Bearer {token} |
- **响应格式**:
  ```json
  {
    "hasCheckedInToday": true
  }
  ```

## 8. 性能与安全要求

### 8.1 性能要求
- **响应时间**：API接口响应时间不超过500ms
- **并发处理**：支持同时1000个用户在线使用
- **数据库性能**：查询响应时间不超过200ms

### 8.2 安全要求
- **认证安全**：使用JWT令牌进行身份验证，令牌有效期合理设置
- **数据安全**：用户敏感信息加密存储
- **接口安全**：所有API接口均需验证令牌
- **网络安全**：使用HTTPS协议传输数据
- **防SQL注入**：使用参数化查询，防止SQL注入攻击

## 11. 项目风险与应对策略

### 11.1 风险分析
- **微信接口变更**：微信API可能会发生变更，影响登录功能
- **数据库性能**：随着数据量增长，数据库查询性能可能下降
- **用户体验**：网络延迟可能影响用户体验

### 11.2 应对策略
- **微信接口适配**：定期关注微信API变更，及时更新代码
- **数据库优化**：使用索引，优化查询语句，考虑分表分库
- **前端优化**：使用缓存，减少网络请求，优化页面加载速度
